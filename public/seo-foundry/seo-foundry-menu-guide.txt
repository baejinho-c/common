SEO Foundry 메뉴 가이드
AI SEO 운영 시스템 (AI SEO Operating System)
================================================================

전체 구조 요약
----------------------------------------------------------------
SEO Foundry는 5개 그룹 / 15개 메뉴로 구성되어 있습니다.
설계 흐름:
  보고(Overview) → 만들고(Content) → AI에 노출(AI Visibility)
  → 키우고 증명(Growth) → 운영(Workspace)

특히 AI Visibility 그룹과 Backlinks↔Citation 연결이
경쟁 도구(Ahrefs, Semrush)와 차별화되는 핵심입니다.


================================================================
1. OVERVIEW (개요) — 한눈에 보고 흐름을 잡는 곳
================================================================

[ Dashboard / 대시보드 ]
- 특징: 전체 프로젝트의 핵심 지표(전체/색인 페이지 수, 노출수,
  클릭수, 평균 CTR·순위)를 KPI 카드와 30일 성장 차트로 요약.
  AI 기회 키워드와 위너 페이지도 함께 표시.
- 필요성: 매일 아침 "지금 우리 SEO가 어떤 상태인가?"를
  30초 안에 파악해야 의사결정이 빨라짐.
- 역할: 경영진/팀 리더가 보는 종합 상황판. 진입점 역할.

[ Workflow / 워크플로우 ]
- 특징: GSC 연결 → 성과 가져오기 → AI 키워드 추천 → 초안 생성
  → 승인 → 발행 → sitemap 갱신 → 성과 추적까지 8단계 파이프라인을
  타임라인으로 시각화. 각 단계 상태와 해당 메뉴 링크 제공.
- 필요성: SEO가 단발성 작업이 아니라 반복되는 자동화 루프임을
  보여줘야 "왜 이 도구가 필요한가"가 명확해짐.
- 역할: 제품 전체를 관통하는 운영 흐름의 지도.


================================================================
2. CONTENT ENGINE (콘텐츠 엔진) — 페이지를 만들어내는 공장
================================================================

[ Projects / 프로젝트 ]
- 특징: 도메인·산업·팀별로 SEO 프로젝트를 분리 관리.
  프로젝트별 페이지 수, 색인율, 클릭수를 테이블로 정리.
- 필요성: 여러 사이트/브랜드를 운영하는 글로벌 SaaS에선
  데이터가 섞이면 안 됨.
- 역할: 모든 작업의 상위 컨테이너(작업 단위 구분).

[ Keywords / 검색어 발굴 ]
- 특징: 산업/주제/지역/제품을 입력하면 AI가 고기회 키워드를 생성.
  의도(거래형·정보형·비교형 등)별 필터, 검색량·난이도·기회 점수 제공.
- 필요성: 모든 SEO의 출발점은 "어떤 검색어를 노릴까".
- 역할: 콘텐츠 제작의 원료 공급원.

[ Topic Map / 토픽 맵 (Content Clusters) ]
- 특징: 흩어진 키워드를 필러 토픽으로 묶어 커버리지(%)·트래픽 비중·
  공백을 시각화. "강함 / 구축 중 / 보강 필요"로 분류.
- 필요성: 2026 SEO는 단일 키워드가 아니라 토픽 권위
  (Topical Authority)로 평가됨. 키워드 단독 관리로는 부족.
- 역할: 콘텐츠를 전략적으로 묶는 설계도.

[ Pages / 페이지 팩토리 ]
- 특징: 수천 개 SEO 페이지를 대량 생성·발행·추적.
  상태(초안/발행/색인/최적화), SEO 점수, 노출·클릭 관리.
  상세 페이지에선 메타데이터·콘텐츠·JSON-LD·AI 제안까지.
- 필요성: 규모(scale)가 프로그래매틱 SEO의 핵심 경쟁력.
- 역할: 실제 결과물을 찍어내는 생산 라인.

3. AI VISIBILITY (AI 가시성) — AI 시대의 차별화 핵심
================================================================

[ AI Visibility / AI 가시성 ]
- 특징: AI 답변 엔진이 내 콘텐츠를 얼마나 자주, 왜 인용하는지 추적.
  AI 인용 점수, 구조화 데이터·FAQ·엔티티·가독성 세부 점수,
  AI Citation Tracker(쿼리별 인용 여부·순위 실시간 모니터링) 포함.
- 필요성: 검색 트래픽이 AI 답변으로 이동 중. "구글 순위"만으론
  미래를 못 봄.
- 역할: 경쟁 도구가 안 쓰는 "AI Citation Tracker" 용어를
  전면에 내세운 핵심 차별화 기능.

[ GEO / 생성형 엔진 최적화 ]
- 특징: ChatGPT·Perplexity·Gemini·Claude 각 엔진별 점유율,
  인용 방식, 최적 콘텐츠 형식, 맞춤 최적화 전략(플레이북)을 따로 제공.
- 필요성: 엔진마다 인용 패턴이 달라 획일적 최적화는 효과 없음.
  AI SEO에 묻히면 가치가 희석됨.
- 역할: 플랫폼별 공략집.

[ llms.txt / 생성기  (New) ]
- 특징: AI 크롤러에게 "내 사이트를 이렇게 읽어라"를 알려주는
  /llms.txt 파일을 자동 생성. 실시간 미리보기·복사·다운로드.
- 필요성: 아직 어떤 주요 SEO 도구도 제대로 안 넣은 블루오션 기능.
- 역할: 단독으로도 검색·바이럴에 유리한 차별화 포인트.

4. GROWTH (성장) — 성과를 키우고 증명하는 곳
================================================================

[ Performance / 성과 분석 ]
- 특징: 전통 지표(노출·클릭·CTR·순위) + AI 레퍼럴 트래픽 별도 집계
  (전환율이 검색보다 약 23배 높음) + 위너 감지
  (확장할 가치가 있는 페이지).
- 필요성: AI 유입을 구분 못 하면 ROI를 증명할 수 없음.
- 역할: 효과를 측정하고 돈으로 환산하는 증거실.

[ Experiments / 실험 ]
- 특징: 제목·메타·스키마·콘텐츠 변경 후 순위/CTR 변화를 A/B로 추적.
  변경 후 AI 인용률 상승(citation lift)까지 연결 측정.
- 필요성: "이 변경이 효과 있었나?"를 데이터로 답해야 함.
- 역할: 가설을 검증하는 실험실.

[ Backlinks / 백링크 ]
- 특징: 백링크 추적, 특히 DA 60+ 고권위 링크 관리.
  각 링크가 AI 인용에 기여했는지 직접 연결해 표시.
- 필요성: AI 인용률을 높이는 5대 요소 중 하나가 고권위 백링크.
  없으면 "왜 인용이 안 되지?"에 답을 못 함.
- 역할: 권위(authority)를 쌓고 Citation Tracker와 연결되는 고리.

[ Competitors / 경쟁사 ]
- 특징: 경쟁사 가시성 벤치마킹, 순위 변동 추적,
  검색어 공백(gap) 발굴.
- 필요성: 내 성과는 항상 상대적임.
- 역할: 시장 내 위치를 파악하는 정찰병.


================================================================
5. WORKSPACE (워크스페이스) — 운영과 설정
================================================================

[ Reports / 리포트 ]
- 특징: 주간/월간/AI SEO 리포트를 예약 자동 생성·이메일 전송.
- 필요성: 이해관계자에게 정기 보고가 필요함.
- 역할: 성과를 외부에 전달하는 창구.

[ Settings / 설정 ]
- 특징: 워크스페이스 정보, 연동(GSC·GA4·OpenAI·Ahrefs),
  알림 관리.
- 필요성: 모든 데이터의 연결 기반이 여기 있음.
- 역할: 제품의 제어판.
